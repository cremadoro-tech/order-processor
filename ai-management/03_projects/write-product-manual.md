# /write-product スキル 運用マニュアル

最終更新: 2026/03/08

---

## 概要

`/write-product` は、商品URLや特徴を入力するだけで、**楽天HTML形式の商品説明文・楽天商品登録CSV・他6チャネル向けコンテンツ**を自動生成するスキルです。

主な機能：
- 競合URLからスペック・訴求ポイントを自動抽出
- 楽天向けHTMLコンテンツを生成・品質チェック
- **楽天商品登録CSV（Shift-JIS）を自動生成**（ジャンルID自動取得つき）
- Amazon / Yahoo! / Shopify / ギフトモール / AuPay / Qoo10 向けに展開

---

## 起動方法

Claude Code のチャット欄に以下の形式で入力する。

```
/write-product
商品名: [商品名・型番]
現在のページURL: [URL]（リニューアルの場合のみ）
競合URL:
  - [メーカーサイトや競合ECのURL 1]
  - [競合URL 2]
  - [競合URL 3]（任意）
変更点・特徴:
  - [特徴や訴求ポイント 1]
  - [特徴や訴求ポイント 2]
価格: [円]
JANコード: [JANコード]（任意）
```

| 項目 | 必須/任意 | 説明 |
|:----|:---------|:----|
| 商品名 | **必須** | 商品名・型番。タイトル生成のベースになる |
| 変更点・特徴 | **必須** | 自社の差別化ポイント・訴求したい特徴 |
| 価格 | **必須** | 税込価格（CSV の販売価格欄に使用） |
| 現在のページURL | 任意 | リニューアル時に既存コンテンツを参照したい場合 |
| 競合URL | 任意 | メーカーサイト・競合ECを2〜3件。精度が上がる |
| JANコード | 任意 | あれば入力。なければカタログIDなし理由=3で自動登録 |

### 入力例

```
/write-product
商品名: TOPスタンプ台 11号 黒・赤・藍・朱
現在のページURL: https://item.rakuten.co.jp/hankoya/top-stamp-11/
競合URL:
  - https://www.shachihata.co.jp/...
  - https://www.tsukioka.co.jp/...
変更点・特徴:
  - 速乾性インク採用。押してすぐ重ねられる
  - スポンジ交換可能で長寿命
  - 全4色展開（黒・赤・藍・朱）
価格: 880
JANコード: （なし）
```

## 実行フロー

### フェーズ 1：楽天コンテンツ生成

```
[インプット受け取り]
      ↓
Step 1: 情報収集（並列実行）
  - Playwright で現在のページ・競合URLにアクセス
  - スペック・訴求ポイント・ターゲットを抽出
      ↓
Step 2: 楽天HTML生成
  - HTMLテーブル形式で商品説明文を生成
  - 必須セクション＋カテゴリ別スペック＋末尾SEOキーワード文
      ↓
Step 3: 品質レビュー（自動）
  - HTML構造・文字数・禁止表現をチェック
  - スコア 80点未満 → 自動修正して再生成
      ↓
Step 4: ユーザーに提示・承認
  - 保存: output/articles/YYYY-MM-DD-[商品名]-rakuten.html
  - 修正がある場合は対応してから次へ
      ↓
Step 5: 楽天商品登録CSVを生成
  - ジャンルIDを自動取得（後述）
  - カタログIDを設定（JANあり/なし対応）
  - 保存: output/articles/YYYY-MM-DD-[商品名]-rakuten.csv
```

### フェーズ 2：他チャネル展開（承認後）

楽天コンテンツをベースに以下を生成：

| チャネル | タイトル | 説明文 | 特記 |
|:--------|:--------|:------|:----|
| Amazon | 150バイト以内 | 箇条書き5点（500字/点） | KW前半配置 |
| Yahoo!ショッピング | 100文字以内 | 2000文字以内テキスト | 価格訴求強調 |
| Shopify / 自社EC | 制限なし | HTML可 | ブランドストーリー重視 |
| ギフトモール | 制限なし | テキスト | ギフト文脈特化 |
| AuPay | 100文字以内 | 2000文字以内テキスト | Yahoo!準拠 |
| Qoo10 | 100文字以内 | シンプルなテキスト | わかりやすい表現 |

## 楽天 HTMLコンテンツの構成

生成されるHTMLテーブルの必須セクション：

| セクション | 内容 | 文字数目安 |
|:---------|:----|:---------|
| 商品名 | 商品名＋ブランド名＋型番＋サイズ | — |
| 商品概要 | 使用シーン＋ギフト訴求 | **150〜250字** |
| 主な特長 | 絵文字つき3〜5個 | **各150〜300字** |
| ブランド | ブランド名＋型番 | — |
| カテゴリ別スペック | 筆記具/印鑑/文具で項目が異なる | — |
| こんな方におすすめ | 職種・シーン・年代 6〜8タイプ | — |
| お手入れ方法 | カテゴリに応じた3〜4項目 | — |
| ギフト・記念品として | 用途別シーン詳細。名入れ対応あれば追記 | — |
| ご注意 | 赤字（`#ba2b2b`）で3〜5項目 | — |
| 末尾SEOキーワード文 | ギフトシーン・用途・イベント名等を羅列 | — |

カテゴリ別スペック項目：
- **筆記具**: インク・替芯 / 機能 / ボール径（芯径）/ 機構 / サイズ / 重量 / 軸仕上
- **印鑑**: 素材 / サイズ / 書体 / 用途（認印・銀行印・実印等）/ 付属品
- **文具**: 素材 / サイズ / 枚数・容量 / カラー展開

## 楽天 CSV 自動生成の仕組み

### ジャンルIDの自動取得

以下の3段階で取得する。成功した時点で次の段階はスキップ。

**① RMS API v2（自動・推奨）**

`scripts/ec/utils/rakuten-genre.js` を使って自店舗商品をタイトル検索し、ジャンルIDを多数決で決定する。

```bash
# 内部的に実行されるコマンド（手動でも使用可）
cd scripts && node ec/utils/rakuten-genre.js "スタンプ台" --verbose
```

出力例：
```json
{"genreId":"111177","genreName":null,"confidence":"rms_item_search"}
```

**② Ichiba Item Search API**（`RAKUTEN_APP_ID` が `.env` に設定されている場合のみ）

楽天市場全体から類似商品を検索し、ジャンルIDを多数決で取得する。

**③ キーワードフォールバックマップ**（①②が失敗した場合）

自社取扱商品カテゴリに特化した固定マッピングで返す。

| カテゴリ | ジャンルID（RMS実測値） |
|:--------|:---------------------|
| スタンプ台・ゴム印 | 111177 |
| 印鑑・認印・銀行印・実印 | 401760 |
| ボールペン | 216081 |
| 万年筆 | 210246 |
| シャープペンシル | 205824 |

> **信頼度が `fallback_map` の場合**: カテゴリマッピングによる推定値なので、RMSアップロード前に確認を推奨。

### カタログIDの設定ルール

| 状況 | CSV に入力する値 |
|:----|:--------------|
| JANコードを提供した場合 | SKU行の「カタログID」にJANコードを入力 |
| JANコードがない場合 | SKU行の「カタログIDなしの理由」に `3` を入力（自社商品・オリジナル） |

### CSV構造

楽天 `normal-item.csv` 形式で出力する。

| 行の種類 | 内容 |
|:--------|:----|
| ヘッダー行 | 公式ヘッダーをそのまま使用 |
| 商品レベル行 | 商品管理番号・商品名・説明文・ジャンルID等 |
| SKUレベル行 | バリエーションごと（色・サイズ等）の価格・在庫・カタログID |

- エンコード: **Shift-JIS（cp932）**
- 改行コード: **CRLF**

## 注意事項・FAQ

**Q: 楽天CSVをRMSにアップロードする前に何を確認すればいい？**

以下の3点を必ず確認する：
1. **ジャンルID** — `confidence` が `fallback_map` の場合は楽天ジャンルページで確認
2. **カタログIDなしの理由=3** — JANコードなし商品に自動設定される。オリジナル商品でなければ変更する
3. **在庫数・価格** — デフォルト在庫は `1`。実際の在庫数に修正する

**Q: ジャンルIDを手動で確認したい**

楽天市場のジャンルページ（[https://ichiba.rakuten.co.jp/category/](https://ichiba.rakuten.co.jp/category/)）からカテゴリを辿り、URLの末尾の数字がジャンルIDになる。

**Q: ジャンルIDツールだけ単体で使いたい**

```bash
cd scripts
node ec/utils/rakuten-genre.js "ボールペン 黒" --verbose
# または
npm run rakuten:genre "印鑑"
```

**Q: 競合URLを指定しなかった場合は？**

変更点・特徴の入力情報のみでコンテンツを生成する。品質は下がる可能性があるため、メーカーサイトのURLを1件以上入れることを推奨。

**Q: バリエーション（色・サイズ展開）がある商品の場合は？**

「変更点・特徴」にバリエーション情報（例: 全4色：黒・赤・藍・朱）を記載すると、CSVのSKU行が自動的にバリエーション数分生成される。
