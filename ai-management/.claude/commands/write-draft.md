# /write-draft — コンテンツ下書きを生成する

## 概要
テーマや指示を受け取り、記事・商品説明・SNS投稿等のコンテンツ下書きを生成するスキル。
生成したファイルは `output/articles/` に保存する。

## 対応コンテンツタイプ

| タイプ | 文字数目安 | 用途 |
|:------|:---------|:-----|
| ブログ記事 | 1500〜3000字 | SEO・情報発信 |
| 商品説明文 | 300〜800字 | EC各チャネルの商品ページ |
| SNS投稿 | 〜140字 | Twitter/X, Instagram等 |
| メルマガ | 500〜1500字 | メール配信 |
| プレスリリース | 800〜1200字 | メディア向け |

## 実行手順

1. **要件を確認する**
   - コンテンツタイプ（上記から選択）
   - テーマ・題材
   - ターゲット読者
   - 目的（認知拡大・購買促進・SEO等）
   - トーン（フォーマル・カジュアル・専門的等）

2. **参考情報を収集する（必要な場合）**
   - `05_learning/` の関連資料を確認する
   - Web検索で最新情報を補足する

3. **下書きを生成する**
   - 読者の課題→解決策の流れで構成する
   - 具体的な数値・事例を含める
   - 見出し構造（H2・H3）を明確にする

4. **ファイルに保存する**
   - 保存先: `output/articles/YYYY-MM-DD-[タイトル].md`
   - ファイルパスをユーザーに伝える

5. **改善提案を行う**
   - タイトル案を3〜5個提示する
   - 追加すると良いコンテンツを提案する

## 商品説明文の場合
- 商品の特徴・ベネフィットを明確に記述
- キーワードを自然に含める（SEO対策）
- 各ECチャネルの文字数制限に合わせてバリエーションを生成する
  - Amazon: 2000文字以内
  - 楽天: 制限なし（詳細に）
  - Yahoo!: 2000文字以内
