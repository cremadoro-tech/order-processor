# /ec-product-design — EC商品ページのデザインSPEC生成

## 概要

EC商品ページ（楽天・Amazon・Yahoo!・Shopify等）のデザイン品質を「AIっぽくない」プロレベルに引き上げるスキル。
競合リサーチ → デザインコンセプト言語化 → PRODUCT-SPEC.md生成 → プロトタイプ生成の流れで、デザイン方向性を確定させる。

**参考:** 梶谷健人「AIだけでデザイン性の高いWebサイトはつくれるのか？8つのポイント」のEC商品ページ応用版。

---

## インプット形式

```
/ec-product-design
商品名: [商品名]
カテゴリ: [印鑑 | 筆記具 | 文具]
ターゲットチャネル: [楽天 | Amazon | Yahoo! | Shopify | 全チャネル]
競合URL:
  - [競合商品ページURL 1]
  - [競合商品ページURL 2]
  - [競合商品ページURL 3]
価格帯: [低価格 | 中価格帯 | 高価格帯・ギフト]
ブランドイメージ: [任意。希望するデザインの方向性]
```

**必須**: 商品名・カテゴリ・ターゲットチャネル
**任意**: 競合URL・価格帯・ブランドイメージ

---

## 実行フロー

### Step 1: 競合リサーチ（ポイント1のEC版）

競合URLが提供されている場合は Playwright で取得。なければ以下のプロンプトで自動リサーチ：

```
以下の商品カテゴリで、{ターゲットチャネル}内のデザイン性が高い商品ページを
5〜8件リサーチしてください。

# 条件
- 商品カテゴリ: {カテゴリ}
- 価格帯: {価格帯}
- 評価の高い（レビュー数が多い）商品を優先
- ページデザインが整理されていて見やすいものを優先

# 分析ポイント
各ページについて以下を報告:
- ファーストビュー（商品画像直下）の構成
- セクション順序と情報の優先度
- 色使い・フォント・余白の特徴
- 差別化されたビジュアル要素
- モバイル表示時の見え方
```

リサーチ結果から3〜5件のベンチマークページをピックアップし、デザインの共通点・優れた点を言語化する。

---

### Step 2: PRODUCT-SPEC.md 生成（ポイント3のEC版）

以下の構造で `output/articles/YYYY-MM-DD-[商品名]-PRODUCT-SPEC.md` を生成する:

```markdown
# {商品名} — 商品ページデザインSPEC

## 1. Design Direction（デザイン方針）

### コンセプト
[一言で表現したデザインの方向性。例: "静謐な格式感 — 伝統×モダンの融合"]

### トーン
[「〜ではなく〜」の対比形式で3〜5項目]
例:
- 「安っぽい派手さ」ではなく「上質な存在感」
- 「情報の羅列」ではなく「ストーリーのある情報設計」
- 「装飾過多」ではなく「意図ある余白」

### カラースキーム
- ベースカラー: [HEX] — [用途]
- サブカラー: [HEX] — [用途]
- アクセントカラー: [HEX] — [CTA・強調用]
- 背景色: [HEX] — [セクション背景]
- テキスト色: [HEX]

### タイポグラフィ
- 見出し: [フォント名] [ウェイト]
- 本文: [フォント名] [ウェイト]
- 強調: [フォント名] [ウェイト]

## 2. セクション構成（上から順）

各セクションに以下を記載:
- セクション名
- 目的（このセクションで何を伝えるか）
- レイアウト（アスキーアートでワイヤーフレーム）
- 背景色・余白の指定
- コピーの方向性

## 3. デザインルール（NEVER / ALWAYS）

### NEVER（絶対にやらない）
[下記「デザイン原則」セクションから該当するものを転記]

### ALWAYS（必ず守る）
[下記「デザイン原則」セクションから該当するものを転記]
```

---

### Step 3: プロトタイプ生成（ポイント2,5のEC版）

PRODUCT-SPEC.mdに基づき、**3パターンの使い捨てHTMLプロトタイプ**を生成する。

```
output/prototypes/
  {商品名}-pattern-A.html  ← パターンA: [コンセプト名]
  {商品名}-pattern-B.html  ← パターンB: [コンセプト名]
  {商品名}-pattern-C.html  ← パターンC: [コンセプト名]
```

**各パターンの方向性:**
- パターンA: SPEC準拠のスタンダード案
- パターンB: より大胆・ビジュアル重視の案
- パターンC: ミニマル・情報整理重視の案

プロトタイプは単体HTMLファイル（CSS埋め込み）として生成し、ブラウザで即確認可能にする。

ユーザーに3パターンを提示 → 採用パターンを選択 → `/write-product` で本番生成に進む。

## デザイン原則

### NEVER — 「AIっぽいデザイン」の回避ルール

以下はEC商品ページで「AIが作った感」が出る典型パターン。**絶対に使わない:**

**レイアウト:**
- 全セクション同じ余白・同じ幅の均一レイアウト
- 情報の優先度なく全て同じフォントサイズで羅列
- テーブルの枠線だけに頼った情報整理（楽天の旧テンプレ感）
- 左右対称で変化のない構成

**色・装飾:**
- 安っぽい虹色グラデーション
- 原色（#FF0000, #0000FF）の多用
- 影（box-shadow）の乱用
- 背景全面に派手なパターン
- 意味のないアイコンの散りばめ

**テキスト:**
- 「！」「★」「♪」の過剰使用
- 全文太字
- 3種類以上のフォントカラーの混在
- 意味のない装飾罫線（━━━━━ など）

**構造:**
- スクロールしても同じパターンの繰り返し
- CTAボタンが5つ以上並ぶ
- 情報量に対してページが不必要に長い

---

### ALWAYS — デザイン品質を上げるルール

**レイアウト:**
- セクション間に明確な余白（padding: 24px〜40px）を確保
- 情報の優先度に応じてフォントサイズ・太さにメリハリ
- 1セクション = 1メッセージの原則
- ファーストビュー（商品画像直下）に最も重要な訴求を配置

**色:**
- ベースカラー + アクセント1色の2色構成を基本とする
- 背景色でセクションを視覚的に分離（白 / 薄いグレー / ブランドカラー薄めの交互）
- CTA（カートボタン周辺）にのみアクセントカラーを使用
- テキストカラーは #333333 以上の濃さ（薄すぎる灰色は禁止）

**タイポグラフィ:**
- 見出しと本文のサイズ比は 1.5:1 以上
- 行間（line-height）は本文で 1.6〜1.8
- 1行あたり最大40文字程度（読みやすさ）
- 強調は太字のみ。色変え + 太字 + 下線の三重強調は禁止

**画像・ビジュアル:**
- 商品画像は高解像度・白背景を基本
- 利用シーン画像を最低1枚含める
- 画像とテキストの間に適切な余白
- スマートフォンでも崩れない画像幅指定（max-width: 100%）

**信頼性・購買促進:**
- レビュー引用・実績数値はセクション上部に配置
- 「ギフトにおすすめ」等の用途提案を含める
- 注意事項は末尾に赤系（#ba2b2b）で簡潔に

## 商品カテゴリ別デザインガイド

### 印鑑・ハンコ・スタンプ

| 要素 | 推奨 |
|:-----|:-----|
| **コンセプト** | 信頼・格式・長く使える品質感 |
| **ベースカラー** | #1a1a2e（ダークネイビー）or #2d2d2d（チャコール） |
| **サブカラー** | #f5f5f0（オフホワイト）or #faf8f5（生成り） |
| **アクセント** | #b8860b（ダークゴールド）or #8b4513（ブラウン） |
| **フォント感** | 明朝体寄りの上品さ。見出しは太め |
| **余白** | 広め。余白で高級感を演出 |
| **キーワード** | 格式、伝統、一生もの、職人、品格 |
| **NG** | ポップすぎる色、カジュアルすぎる表現、安売り感 |

### 筆記具（ボールペン・万年筆・シャープペンシル等）

| 要素 | 推奨 |
|:-----|:-----|
| **コンセプト** | 洗練・クラフトマンシップ・書く喜び |
| **ベースカラー** | #2c3e50（ダークブルーグレー）or #1a1a1a（ほぼ黒） |
| **サブカラー** | #ecf0f1（ライトグレー）or #ffffff |
| **アクセント** | #e67e22（オレンジ）or #2980b9（ブルー） |
| **フォント感** | ゴシック体ベース。クリーンで現代的 |
| **余白** | 標準〜やや広め |
| **キーワード** | 書き心地、精密、デザイン、ギフト、ビジネス |
| **NG** | 子供っぽい色使い、チープな装飾 |

### 文具（ノート・ファイル・テープ・付箋等）

| 要素 | 推奨 |
|:-----|:-----|
| **コンセプト** | 親しみ・日常の質の向上・使いやすさ |
| **ベースカラー** | #ffffff or #fafafa |
| **サブカラー** | #f0ead6（ベージュ系）or #e8f4f8（水色系） |
| **アクセント** | #ff6b6b（コーラル）or #4ecdc4（ティール） |
| **フォント感** | 丸ゴシック or ゴシック。親しみやすさ |
| **余白** | 標準。詰まりすぎず空きすぎず |
| **キーワード** | 便利、かわいい、整理整頓、毎日使い、コスパ |
| **NG** | 高級ブランド風の演出、堅すぎる表現 |

## チャネル別デザイン制約

### 楽天市場

**技術制約:**
- HTMLタグ使用可（ただしJavaScript不可）
- インラインCSS推奨（`<style>`タグも可だが一部環境で無効）
- PC版とSP（スマートフォン版）は**別HTML**で管理
- 画像は楽天GOLD（R-Cabinet）にアップロード
- iframe不可、外部CSS不可

**デザイン方針:**
- PC版を先に完成 → SP版を一括生成（ポイント6）
- 情報量は多くてOKだが、セクション分離と余白で整理する
- テーブルレイアウトからdiv+インラインCSSレイアウトへ移行
- 横幅は PC: 750px基準、SP: 100%（max-width制約）

### Amazon

**技術制約:**
- A+コンテンツ（旧EBC）: モジュール選択式
- 使用可能モジュール: Standard Image, Text, Comparison Chart 等
- HTMLタグは使用不可（テキストのみ or モジュール）
- 商品タイトル150バイト以内、箇条書き5点

**デザイン方針:**
- ブランドストーリー → 商品特長 → 比較表の流れ
- 画像で魅せる（テキストは補助）
- 白背景の高品質商品画像が最重要

### Yahoo!ショッピング

**技術制約:**
- HTML使用可（限定的）
- テキスト2,000文字以内
- 画像はストアクリエイターPro経由

**デザイン方針:**
- シンプル＆信頼感重視
- 価格訴求・ポイント訴求を上部に

### Shopify / 自社EC

**技術制約:**
- HTML/CSS/JS フル対応
- Liquid テンプレートエンジン
- レスポンシブ対応必須

**デザイン方針:**
- ブランド世界観を全面に表現
- 記事のポイント8（Next.js + Tailwind + Framer Motion）の技術スタック適用可
- 商品詳細はアコーディオンUIで整理

## `/write-product` との連携

このスキルで生成した PRODUCT-SPEC.md は、`/write-product` 実行時に自動参照される。

**推奨ワークフロー:**
1. `/ec-product-design` でデザインSPEC + プロトタイプ生成
2. ユーザーがプロトタイプから採用パターンを選択
3. `/write-product` で採用パターンのSPECに基づいて本番コンテンツ生成

PRODUCT-SPEC.mdが既に存在する場合、`/write-product` はそのSPECのデザインルールに従って楽天HTMLを生成する。
