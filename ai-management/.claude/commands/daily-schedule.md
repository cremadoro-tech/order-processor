# /daily-schedule — 毎朝の工程表を生成する

## 概要
毎朝の作業工程表を自動生成するスキル。Google Calendar の予定、GitHub Issues のタスク、前日の未完了タスク、**要返信メール**を統合して、15分刻みの最適なスケジュールを作成する。

## 実行手順

### Step 1〜1.5: 情報収集（並列実行）

以下を**並列**で実行する：

**Step 1: Google Calendar から今日の予定を取得**
- MCP `google-calendar` の `gcal_list_events` を使用して本日の全予定を取得
- 予定の時間・場所・内容を確認する

**Step 1.5: Gmail から要返信メールを確認**
- MCP `Gmail` の `gmail_search_messages` で直近72時間の未読メールを取得
  - クエリ: `is:unread newer_than:3d -category:promotions -category:social -category:updates -category:forums`
  - `maxResults: 30`
- 各メールを `gmail_read_message` で本文を取得（最大10件まで）
- AIが「返信が必要か」を以下の基準で判定：
  - **返信が必要**: 質問・依頼・確認事項・アクション要求が含まれる業務メール
  - **除外**: システム通知（楽天/Amazon/GitHub等）、メルマガ、広告、自動返信、info@系の送信専用アドレス
- 返信が必要なメールについて：
  1. `gmail_read_thread` でスレッド全体を取得し文脈を把握
  2. `gmail_create_draft` で返信下書きを作成（`threadId` を指定して返信スレッドに紐づけ）
  3. 下書きの文体は丁寧かつ簡潔なビジネス日本語（`preferences.md` 参照）
- **重要**: 下書きはあくまで「たたき台」。送信はせず、Gmailの下書きフォルダに保存するのみ

### Step 2: GitHub Issues からタスクを取得
- MCP `github` を使用して `status:open` のIssueを取得
- ラベル `priority:high` のタスクを最優先とする

### Step 3: DAILY.md から昨日の未完了タスクを確認
- `~/My WorkSpace/ai-management/DAILY.md` を読み込む
- チェックされていない `[ ]` タスクを抽出する

### Step 4: タスクの分類と優先度付け
- 緊急度（締め切り・約束）× 重要度（事業インパクト）でマトリクス評価
- 認知負荷（集中力が必要か）を考慮してブロック配置
- 集中作業は午前、MTG・連絡は午後に配置する
- **要返信メール**は工程表の「メールチェック」枠に統合する

### Step 5: 15分刻みの工程表を生成
```
09:00-09:15  メール・Slack確認
09:15-10:30  【集中作業】優先度Highタスク
10:30-10:45  休憩
...
```

### Step 6: DAILY.md に書き込む
- 本日の日付セクションを追加
- 生成した工程表を書き込む
- 今日のゴール（Top 3）を設定する

## 出力フォーマット

```markdown
## YYYY/MM/DD（曜日）の工程表

### 今日のゴール（Top 3）
1.
2.
3.

### カレンダー予定
| 時間 | 内容 |
|:----|:-----|
| XX:XX | |

### 要返信メール
| # | 差出人 | 件名 | 緊急度 | 下書き |
|:--|:------|:-----|:------|:------|
| 1 | 名前 | 件名 | 高/中/低 | 作成済み |

> 下書きはGmailの下書きフォルダに保存済み。確認・編集してから送信してください。

### 工程表（15分刻み）
| 時間 | タスク | 備考 |
|:----|:------|:----|
| 09:00 | | |

### 本日のタスクリスト
- [ ] 優先度High
- [ ] 優先度Medium
```

## 完了後
工程表を表示してユーザーに確認を促す。
- 要返信メールがあった場合は件数と下書き作成状況を報告する
- 修正希望があれば反映する
