# /aupay-ad-strategy — AuPay Market向け広告・キャンペーン戦略を自動生成する

## 概要

商品情報と予算をインプットとして受け取り、AuPay Market（旧WowMa）特有の
三太郎の日・Pontaポイント倍率・クーポン戦略を軸にした広告・キャンペーン計画を
4ステップで自動生成するスキル。

- **Step 1〜4**: Claude 本体（外部ツール不要）
- **競合調査オプション**: Playwright + Nint（任意）

---

## インプット形式

```
/aupay-ad-strategy
商品カテゴリ: [例: おむつスタンプ / ボールペン / 認印 / 実印]
対象月: [例: 2026-03 / 2026-04]（省略時は当月）
月間広告予算: [例: 5,000円 / 10,000円 / 未定]
訴求ポイント: [例: 低価格・補充インク付き・ギフト向け]（複数可）
現在のAuPay売上状況: [例: 月3〜5件 / 月10件以上 / 新規出品]（任意）
競合調査: [あり / なし]（省略時は「なし」）
```

**必須:** 商品カテゴリ・対象月
**任意:** 月間広告予算・訴求ポイント・現在の売上状況・競合調査

---

## 実行フロー

```
[インプット受け取り]
       ↓
[Step 1] AuPay Market特性分析 + 三太郎の日カレンダー生成
       ↓

       ↓
[ファイル保存] output/articles/YYYY-MM-DD-aupay-ad-strategy-[カテゴリ英名].md
```

## Step 1: AuPay Market特性分析 + 三太郎の日カレンダー

**ツール:** Claude本体

### 三太郎の日分析

対象月の三太郎の日（毎月3日・13日・23日）について以下を出力する：

```
■ [月]の三太郎の日スケジュール

【3日】
- 曜日: [曜日]
- 前日準備: [2日にすべきこと]
- 当日施策: [ポイント設定・クーポン・価格調整]
- 終了後の動作: [在庫補充・レビュー確認など]

【13日】
（同上フォーマット）

【23日】
（同上フォーマット）
```

### AuPay Market特性チェックリスト

以下の観点で商品カテゴリと対象月を分析する：

| 観点 | 内容 |
|:----|:-----|
| Pontaポイント感度 | 当該カテゴリのユーザーがポイント倍率を重視する度合い |
| 季節需要 | 対象月における当該商品の需要ピーク・谷 |
| 入園・新生活需要 | 3〜4月の特需が見込める商品か |
| ギフト需要 | 出産祝い・入学祝いなどギフト用途が見込めるか |
| 競合密度 | AuPay Market内の競合商品数（多/中/少） |

## Step 2: Pontaポイント倍率戦略 + クーポン設計

**ツール:** Claude本体

### Pontaポイント倍率シミュレーション

月間予算と目標売上から最適なポイント倍率を試算する。

```
■ ポイント倍率設計

【通常時（三太郎の日以外）】
推奨倍率: [N]倍
理由: [コスト負担 vs 購買促進効果のバランス]
追加コスト目安: 売上の約[X]%

【三太郎の日（3/13/23日）】
推奨倍率: [N]倍（通常の[N]倍）
理由: プラットフォーム増量日に合わせた上乗せで上位表示狙い
追加コスト目安: 売上の約[X]%

【特別月（年間で高需要が見込まれる月）】
推奨倍率: [N]倍
例: 入園準備(3-4月)・卒業入学ギフト(2-3月)など
```

### クーポン設計

```
■ クーポン戦略

【三太郎の日用クーポン】
割引率: [X]%OFF または [X]円引き
期間: [三太郎の日 当日のみ / 前後3日間 / 週末セット]
配布条件: [全商品 / 一定金額以上 / 初回購入者]
期待効果: CVR +[X]% / リピート購入促進

【通常期クーポン】
割引率: [X]%OFF
目的: カート落ち防止 / 比較離脱防止

【セット購入クーポン】
設計: [例: 2個購入で10%OFF]
目的: 客単価アップ / LTV向上
```

## Step 3: 商品別広告コピー生成（AuPay特化）

**ツール:** Claude本体

AuPay Market特有の訴求軸に合わせて以下のコピーセットを生成する。

### AuPay特化の訴求軸（優先度順）

1. **Pontaポイント訴求** → 「ポイントでもっとお得」「ポイ活に最適」
2. **三太郎の日連動** → 「毎月3・13・23日はさらにお得」
3. **価格比較訴求** → 「〜円以下」「コスパ最強」「送料無料」
4. **ランキング訴求** → 「ランキング入賞」「人気商品」
5. **ギフト需要訴求** → 「プレゼントに最適」「入園祝い」（季節限定）

### 生成するコピーセット

```
■ タイトルコピー（100文字以内 × 3案）

[案A] Pontaポイント訴求型:
[商品名] + ポイント倍率 + 主要スペック + 送料無料

[案B] 価格訴求型:
[価格帯] + [商品名] + 主要スペック + コスパキーワード

[案C] ギフト/シーン訴求型:
[利用シーン] + [商品名] + 主要スペック + ギフトキーワード

■ 三太郎の日専用バナーコピー（短文 × 2案）

[案1]: [商品名]が三太郎の日にポイント[N]倍！[価格]でお得にゲット
[案2]: [利用シーン]に！三太郎の日は[商品名]をポイントでおトクに

■ 商品説明文 冒頭フック（200文字以内）

Pontaポイントが貯まる・使えるauPAY マーケット限定の訴求を含めた
説明文の冒頭フックを生成する。

■ クーポン訴求バナー文言（50文字以内）

三太郎の日クーポン・通常期クーポン用の短文コピー。
```

## Step 4: 月間キャンペーンカレンダー + 予算配分表

**ツール:** Claude本体

### 月間アクションカレンダー

```
■ [YYYY年MM月] AuPay Market キャンペーンカレンダー

【月初（1〜2日）】
□ 前月実績確認・レビュー返信
□ クーポン有効期限の確認・更新
□ 3日（三太郎の日）に向けた在庫確認

【三太郎の日 前日（2日・12日・22日）】
□ ポイント倍率設定の確認
□ 三太郎の日用クーポン有効化
□ 商品ページのキャッチコピーを三太郎の日バージョンに更新
□ 在庫が十分にあるか最終確認

【三太郎の日 当日（3日・13日・23日）】
□ 出品ページの表示確認（クーポン・ポイント表示）
□ 注文増加への対応体制確認
□ 競合の価格・施策をモニタリング

【三太郎の日 翌日（4日・14日・24日）】
□ 注文数・CVR・売上を記録
□ レビュー依頼メッセージ送信
□ クーポンを通常設定に戻す

【月末（25〜31日）】
□ 翌月の三太郎の日作戦を計画
□ 月間売上・広告効果の集計
□ ポイント倍率・クーポンの振り返り改善
```

### 予算配分表

```
■ 月間広告予算配分（合計: [X]円）

| 施策 | 配分 | 金額 | 目的 |
|:----|:----|:----|:----|
| ポイント倍率上乗せ（通常時） | 40% | X円 | 安定売上・検索表示維持 |
| ポイント倍率上乗せ（三太郎の日） | 35% | X円 | 集中的な売上最大化 |
| クーポン原資 | 15% | X円 | CVR向上・カート落ち防止 |
| 予備（突発対応） | 10% | X円 | 競合対抗・緊急値下げ等 |

■ ROI目標設定

三太郎の日の売上目標: 通常日の[N]倍
月間売上目標: [X]件 / [X]円
ポイント施策コスト回収目標: [X]ヶ月
```

## エラーハンドリング

| 状況 | 対応 |
|:----|:----|
| 月間予算が未指定の場合 | 予算なし前提でポイント倍率・クーポンのみ設計する |
| 対象月が過去の場合 | 警告を出して当月で続行するか確認する |
| 商品カテゴリが不明瞭な場合 | 「印鑑・筆記具・文具」の3カテゴリのどれに近いか確認してから開始する |
| 競合調査ありでPlaywright失敗の場合 | 「競合調査をスキップして続行するか？」を確認し、Claude本体での推定分析に切り替える |
