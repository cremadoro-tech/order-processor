# /issue-triage — タスクをGitHub Issueに自動登録する

## 概要
ユーザーのテキスト入力（自然言語）から GitHub Issue を自動作成するスキル。
タスクの種類・優先度・カテゴリを自動判定し、適切なラベルを付与して登録する。

## 実行手順

1. **入力テキストを解析する**
   - タスクの内容・目的・期限を把握する
   - 以下の判定を行う：
     - タイプ: `task`（作業） / `idea`（アイデア） / `content`（コンテンツ作成）
     - 優先度: `high`（今週中）/ `medium`（今月中）/ `low`（いつか）
     - カテゴリ: `ec`（EC業務）/ `content`（コンテンツ）/ `strategy`（戦略）/ `finance`（経理）/ `admin`（管理）

2. **既存 Issue との重複チェック**
   - MCP `github` を使用して open Issues を検索
   - 類似タスクがある場合はユーザーに確認する

3. **GitHub Issue を作成する**
   - タイトル: 簡潔に20文字以内
   - 本文: タスク詳細・背景・完了条件を記述
   - ラベル: 判定したタイプ・優先度・カテゴリを付与
   - Milestone: 期限がある場合は設定

4. **作成結果を報告する**
   - Issue番号・URL・タイトルを表示
   - 「他に登録するタスクはありますか？」と確認

## ラベル定義

| ラベル | 説明 |
|:------|:-----|
| `type:task` | 具体的な作業タスク |
| `type:idea` | アイデア・検討事項 |
| `type:content` | コンテンツ作成 |
| `priority:high` | 今週中に対応 |
| `priority:medium` | 今月中に対応 |
| `priority:low` | いつか対応 |
| `ec` | EC業務（仕入れ・出品・在庫等） |
| `content` | コンテンツ・マーケティング |
| `strategy` | 戦略・経営企画 |
| `finance` | 経理・財務 |
| `admin` | 管理・運用 |
