# /ec-strategy — EC物販向けマーケ戦略・LP構成案を6ステップで自動生成する

## 概要

商品カテゴリ・価格帯・購買シーンをインプットとして受け取り、
感情的摩擦の抽出から競合調査・ポジション確定・コピー生成・LP構成案・LTV設計まで
6ステップで完結するEC物販向けマーケティング戦略を自動生成するスキル。

- **Step 1〜3**: Claude 本体（外部ツール不要）
- **Step 2**: Nint（Playwright）+ 競合ページスクレイプ（Playwright）+ NotebookLM（任意）
- **Step 4〜6**: Claude 本体（前ステップの出力をチェーン受け渡し）

---

## インプット形式

```
/ec-strategy
商品カテゴリ: [例: ボールペン / 印鑑 / 文具ギフトセット]
主力価格帯: [例: 500〜3000円]
購買シーン: [例: ギフト用途 / 自分用日常使い / 法人需要]（複数可、スラッシュ区切り）
競合商品URL: [任意: Amazon/楽天の競合商品URL 1〜3件]
モード: [全自動 / ステップ確認]（省略時は「ステップ確認」）
```

**必須:** 商品カテゴリ・主力価格帯・購買シーン
**任意:** 競合商品URL・モード

---

## 実行モード

| モード | 動作 |
|:------|:-----|
| ステップ確認（デフォルト） | 各ステップ完了後にユーザーの確認を取り、承認後に次へ進む |
| 全自動 | Step 1〜6 を連続実行し、完了後に全結果を提示してファイル保存する |

## Step 1: 感情的摩擦30個抽出

**ツール:** Claude 本体のみ

以下の商品情報をもとに、顧客が購入前後に感じる感情的摩擦を30個抽出する。

```
購買プロセスの5フェーズに分けて、各フェーズで6個ずつ（合計30個）を抽出する。

■フェーズ1「認知・発見」（購入を思い立つ段階）
 → このカテゴリ・価格帯・シーンで顧客が最初に感じる不安・迷いを6個

■フェーズ2「比較・検討」（複数商品を比べている段階）
 → 競合と比較する際に感じる疑問・ためらいを6個

■フェーズ3「購入直前」（カートに入れてから決済前）
 → 最後の一歩を踏み出せない理由・カート落ちを引き起こす感情を6個

■フェーズ4「受取・開封」（商品が届いた瞬間）
 → 期待と現実のギャップから生まれる不満・失望・驚きを6個

■フェーズ5「使用後・リピート検討」（使い始めてしばらく後）
 → 継続使用・再購入・他者への推薦を妨げる感情を6個

【条件】
・「価格が高い」「レビューが少ない」等の表面的な不安は除外する
・「ギフトで渡して相手の反応が薄くて恥をかいた」「安物買いの銭失いを繰り返している気がして自己嫌悪」
  のような、具体的な体験に紐づいた感情レベルの摩擦を抽出する
・各摩擦に「顧客の内なる声（具体的なセリフ）」を1行付記する
・各摩擦に「商品ページ・コピーで解消できる対処の方向性」を1行付記する
```

## Step 2: 競合調査

**ツール:** Playwright（Nint・競合ページ）+ Claude分析 + NotebookLM（任意）

### Sub-step A: Nint で売れ筋ランキングを取得

MCP `playwright` を使用して Nint にログインし、指定カテゴリの売れ筋 TOP10 を取得する。

```
- URL: https://www.nint.jp
- 認証: .env の NINT_EMAIL / NINT_PASSWORD を使用
- 取得データ: 商品名・価格・順位・前週比・推定月商（取得できる場合）
- 絞り込み: 入力された「商品カテゴリ」でジャンル検索
- Nintへのログインが失敗した場合: ユーザーに手動ログインを依頼し、完了を待ってから継続する
```

### Sub-step B: 競合商品ページをスクレイプ

MCP `playwright` を使用して競合商品ページを取得・解析する。

```
対象URL（優先順）:
  1. Sub-step A で特定した Nint 売れ筋 TOP3 の商品URL
  2. ユーザーが入力した「競合商品URL」（ある場合は必ず含める）

取得・解析する項目:
  - 商品タイトル（全文）
  - 価格・ポイント設定・割引表示
  - 商品説明文・キャッチコピー（冒頭500文字）
  - 特長・箇条書き（Amazonの場合：bullet points 5点）
  - 楽天の場合：ページ構成（セクション見出しを順に抽出）
  - レビュー数・平均評価点
  - 上位レビュー5件の要点（評価高・評価低 各2〜3件）
  - 画像の ALT テキスト（画像枚数と構成を推定）

スクレイプできない場合: その商品はスキップして次の候補に進む
```

### Sub-step C: NotebookLM 分析（任意）

事前に NotebookLM へ競合ページをノートブック登録済みの場合のみ実行する。
登録されていない場合はこの sub-step をスキップし、Sub-step D に進む。

```
実行コマンド:
  python .agents/skills/notebooklm/scripts/run.py ask_question.py \
    --question "「{商品カテゴリ}」カテゴリの競合商品について、USP・差別化軸・顧客レビューで繰り返し言及される不満を分析してください"

取得した回答は Sub-step D の Claude 分析インプットとして使用する。
```

### Sub-step D: Claude 統合分析

Sub-step A〜C で収集したデータを統合し、以下を分析する。

```
・競合が共通して押さえている訴求軸（この軸での差別化は難しい）
・競合が見落としている / 解決できていない摩擦（Step 1 の摩擦リストと照合）
・価格帯・レビュー数・評価の分布
・タイトル戦略のパターン（頻出キーワード・構文）
・ページ構成の共通パターンと特徴的な差異
```

```markdown

### 競合分析サマリー
- 競合が共通して押さえている訴求軸（差別化が難しいポイント）:
```

## Step 3: 自社ポジション確定・購買導線マップ設計

**ツール:** Claude（Step 1+2 の統合分析）

Step 1 の「感情的摩擦 TOP5」と Step 2 の「競合が見落としている摩擦」を照合し、
競合が手を付けていない、かつ顧客が最も困っている領域を自社ポジションとして確定する。

```
【ポジション確定の手順】
1. Step 1 の重大摩擦 TOP5 と、Step 2 の競合が解決できていない摩擦を照合する
2. 両方に登場する「未解決の重大摩擦」を特定する
3. その摩擦を自社が最も効果的に解消できる訴求軸を USP として定義する
4. 競合ポジションマップ（価格帯 × 訴求軸の2軸）上に自社の位置を確定する

【購買導線マップの設計】
5段階 × 6タッチポイント = 30マスを設計する

段階: 無関心→認知 / 興味・関心 / 比較・検討 / 購入意思確定 / 購入後・リピート
タッチポイント: 検索/流入 / 商品一覧・サムネ / ページ冒頭FV / ページ本文 / カート〜決済 / 購入後

各マスに定義するもの:
  ・顧客の状態（感情・思考・行動）
  ・解消すべき摩擦（Step 1 から引用）
  ・打ち手（見せるコンテンツ・コピー・仕掛け）
  ・KPI（このタッチポイントの成功指標）
```

## Step 6: LTV最大化施策設計

**ツール:** Claude（Step 1 フェーズ5 + Step 3 の購入後導線を活用）

```
【3領域での設計】

■領域1: アップセル設計（購入単価を上げる）
・上位グレード・名入れ対応版・バンドル商品のラインアップ案 × 5案
  （各案に価格帯・利益率・想定購入率を付記）
・各アップセル商品の提案タイミングと使用するコピー（Step 4 から引用・追加生成）
  （商品ページ内 / カート画面 / 購入後メール の3タイミング）

■領域2: クロスセル設計（購入点数を増やす）
・カテゴリ内クロスセルマップ（購入商品 → 提案商品の対応表）× 5案
  （例: ボールペン購入→替芯・ペンケース、印鑑購入→印鑑ケース・朱肉）
・各案に「なぜフロント購入者が欲しくなるか」の心理的理由を付記
・バンドル価格設定の方針（単品合計比 〇〇%オフ等）
・クロスセルを見せるタイミングと場所

■領域3: リピート・ファン化設計（再購入率を上げる）
・購入後フォローメール設計（D+1 / D+7 / D+30 の3通）
  各メールに: 送信タイミング・目的・件名案・本文の構成・CTA
・レビュー獲得施策（依頼タイミング・依頼文コピー・インセンティブ設計）
・SNS誘導設計（購入者をInstagram・X フォロワーに転換する仕掛け）
・ポイント・クーポンを使ったリピート設計（楽天SPU・Yahoo!ストアポイント活用）

【LTV計算フレームワーク】
・現状想定LTV = 購入単価 × 年間購入回数 × 継続年数
・施策後の目標値と改善率を試算する
```

**出力形式:**
```markdown
## Step 6: LTV最大化施策設計

### 現状 LTV 診断
- 購入単価（想定）:
- 年間購入回数（想定）:
- 継続年数（想定）:
```

## 注意事項

- **Nint ログイン失敗時**: ユーザーに手動ログインを依頼し、完了を待ってから Sub-step B に進む
- **競合ページスクレイプ失敗時**: その商品をスキップして次の候補に進む。全候補が失敗した場合は Sub-step B なしで Sub-step D（Claude分析のみ）に進む
- **NotebookLM 未登録時**: Sub-step C をスキップして通知する
- **数値目標の厳守**: Step 4 の 100 案は必ず100案生成する（内訳の数値を満たすこと）
- **チェーン設計**: 後続ステップほど前ステップの具体的な出力（摩擦番号・コピー案番号等）を引用する
- **業界特化**: 文具・印鑑ECの業界知識を活かした具体的な内容にする（汎用的な抽象論は避ける）
